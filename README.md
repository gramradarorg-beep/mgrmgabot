# MGRMGA Trade Bot

Постит покупки и продажи MGRMGA в Telegram-канал с весёлым сопровождением
(бодрим на покупках, успокаиваем на продажах). Данные — из TonAPI, отправка — через Telegram Bot API.
Без внешних зависимостей, чистый Node ≥18.

## Как работает
1. Каждые N секунд опрашивает `TonAPI /accounts/{POOL}/events` по адресу пула MGRMGA/TON.
2. Находит действия `JettonSwap`, определяет покупку (TON → MGRMGA) или продажу (MGRMGA → TON).
3. Берёт случайную фразу нужного тира (по объёму сделки в TON) и постит в канал.
4. Помнит отправленные сделки в `state.json`, дубли не шлёт. Старую историю при первом
   запуске молча помечает как «видел» — канал не засыпается прошлыми сделками.

## Настройка
1. Создай бота у **@BotFather**, получи `TELEGRAM_BOT_TOKEN`.
2. Добавь бота **администратором** в канал (право «Публикация сообщений»).
3. Ключ TonAPI — на **tonconsole.com → TonAPI** (`TONAPI_KEY`).
4. `cp .env.example .env` и заполни поля.
5. `POOL_ADDRESS` — адрес пула MGRMGA/TON на STON.fi. Появится **после создания пула**;
   до этого бот в боевом режиме не стартует (нечего слушать).

## Запуск
```bash
node index.mjs          # боевой цикл
node index.mjs --test   # отправить тестовую покупку и продажу в канал (проверка токена/прав/вёрстки)
```

Проверить оформление можно **прямо сейчас** через `--test` — нужны только `TELEGRAM_BOT_TOKEN`
и `CHANNEL_ID`. Пул и TonAPI для теста не требуются.

## Держать 24/7
- Проще всего — на том же сервере, где сайт. Через `pm2`:
  ```bash
  npm i -g pm2
  pm2 start index.mjs --name mgrmga-bot
  pm2 save
  ```
- Либо `node index.mjs` в screen/systemd.

## Правка фраз
Все тексты — в `phrases.mjs` (тиры `small / medium / large / whale` для покупок,
`small / medium / large / dump` для продаж). Пороги тиров в TON — там же в `TIERS`.
